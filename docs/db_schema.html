Известные проблемы и ограничения
⚠ Дубликаты структур

NaCl (id=3 synthetic + id=22 COD) и MgO (id=19 + id=25). COD хранит формулы алфавитно («Cl Na»), дедупликация по формуле не срабатывает. Нужно удалить синтетические записи без координат.

⚠ Неверные типы у Fe и W

Записаны как monoclinic, должны быть cubic. Ошибка вошла при ручном заполнении через complete_db.py.

⚠ 20 структур без координат

Синтетические записи id=2–21 (UO₂, Fe, Al, Ti, UN …) имеют метаданные, но structure_site пустой. Поиск по ним невозможен до загрузки XYZ-файлов из COD.

⚠ Изоструктурная неразличимость

NaCl, CaO, MgO, KCl — одинаковая rock-salt структура, одинаковые norm_x/y/z. COORD_MATCH не различает их без учёта атомных символов. Побеждает тот, у кого больше строк в structure_site.

⚠ fract_x/y/z — название неточное

Хранят абсолютные координаты в Å, а не дробные [0,1]. Нормализация делается отдельно в norm_x/y/z. На работу алгоритма не влияет.

⚠ CatBoost — только 3 класса

Модель знает cubic_f, cubic_p, hex_p. Confidence всегда ~0.33 (равномерное). Нужна переобучка на полном датасете 8 систем. Интеграция с БД готова.

✓ feature_vector_cache готов к использованию

Схема правильная, таблица пустая. Подключить — одна строка в ml_predict.py после вычисления вектора.

ℹ NOT NULL везде

Все поля NOT NULL. Пустые значения хранятся как '', {} или [] в JSONB. Упрощает обработку в Python: нет проверок на None для строковых полей.

ℹ ON DELETE CASCADE vs SET NULL

CASCADE у структурных таблиц: удалить reference_structure → каскадно удалятся structure_site и substance_info. SET NULL у recognition_result: исторические результаты не теряются при удалении структуры из справочника.
